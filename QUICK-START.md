## 📋 Interview Structure Recommendation

### Part 1: Theoretical Knowledge (30-45 minutes)
Use questions from `React-Next-Interview-Guide.md`:

**React.js (Choose 3-4 questions):**
- Component lifecycle & hooks
- Performance optimization strategies
- Form handling patterns
- Error boundaries

**Next.js (Choose 3-4 questions):**
- Data fetching methods (SSR vs SSG vs ISR)
- Image optimization
- Authentication patterns
- Pages Router vs App Router

### Part 2: Practical Problem-Solving (30-45 minutes)
Use scenarios from `React-Next-AI-Scenarios.md`:

**Present one scenario:**
- Scenario 5A: React infinite re-render bug
- Scenario 5B: Next.js hydration mismatch
- Scenario 6: TypeScript type safety issues

**Ask candidate to:**
1. Identify all issues in the code
2. Use AI assistant to help fix them
3. Explain their approach and reasoning
4. Validate AI suggestions critically

## 🎯 Difficulty Levels

### Junior/Mid-Level Focus
- Basic React hooks (useState, useEffect)
- Controlled components
- Simple Next.js routing
- Basic data fetching

### Senior-Level Focus (Recommended)
- Advanced hooks patterns (useMemo, useCallback, custom hooks)
- Performance optimization strategies
- ISR and caching strategies
- Authentication & middleware
- Server Components (App Router)
- TypeScript integration

### Staff/Principal-Level Additions
- Concurrent features
- Advanced state management architecture
- Micro-frontends integration
- Performance profiling and optimization
- Security hardening

## 💡 Pro Tips for Interviewers

1. **Don't just ask for definitions** - Ask "why" and "when" questions
   - ❌ "What is useEffect?"
   - ✅ "When would you use useEffect vs useLayoutEffect?"

2. **Present real scenarios** - Use the debugging scenarios to assess practical skills

3. **Evaluate AI collaboration** - Watch how they:
   - Craft prompts
   - Iterate on responses
   - Validate suggestions
   - Combine AI output with their knowledge

4. **Look for depth** - Senior developers should explain:
   - Trade-offs between approaches
   - Performance implications
   - Edge cases and failure modes
   - Best practices and why they exist

5. **Test debugging methodology** - Do they:
   - Reproduce the issue first?
   - Isolate the problem?
   - Test fixes incrementally?
   - Consider side effects?
